Bybit API Endpoints Debug

Эти URL используются CCXT для получения данных с Bybit API:

🔍 Open Orders (fetchOpenOrders)

GET
https://api.bybit.com/v5/order/realtime
Получение открытых ордеров для UTA аккаунтов
GET
https://api.bybit.com/unified/v3/private/order/unfilled-orders
Альтернативный эндпоинт для открытых ордеров

📈 My Trades (fetchMyTrades)

GET
https://api.bybit.com/v5/execution/list
Получение истории сделок для UTA аккаунтов
GET
https://api.bybit.com/unified/v3/private/execution/list
Альтернативный эндпоинт для истории сделок

📋 All Orders (fetchOrders)

GET
https://api.bybit.com/v5/order/history
История ордеров (не поддерживается для UTA аккаунтов после 5/02)
GET
https://api.bybit.com/v5/order/realtime
Открытые ордеры (fallback)

💰 Account Balance

GET
https://api.bybit.com/v5/account/wallet-balance
Баланс кошелька UTA аккаунта

🔧 Параметры запросов

Обязательные параметры:

  • category - тип торговли (spot, linear, inverse, option)
  • symbol - торговая пара (опционально для некоторых запросов)
  • limit - количество записей (по умолчанию 20, максимум 50)

Аутентификация:

  • X-BAPI-API-KEY - API ключ
  • X-BAPI-TIMESTAMP - временная метка
  • X-BAPI-SIGN - подпись запроса
  • X-BAPI-RECV-WINDOW - окно получения (по умолчанию 5000ms)

❌ Известные проблемы

  • UTA аккаунты: fetchOrders() не поддерживается после 5/02/2024
  • Символы: Некоторые методы требуют указания конкретного символа
  • Категории: Нужно указывать правильную категорию (spot/linear/etc)
  • Лимиты: API имеет ограничения по частоте запросов
