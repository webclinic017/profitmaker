## Использование в виджете Data Provider Setup

### 1. Создание CCXT Server провайдера

1. Откройте виджет **Data Provider Setup**
2. В поле **Provider Type** выберите **CCXT Server** (статус: Implemented)
3. Заполните обязательные поля:
   - **Provider Name**: Имя провайдера (например, "Main CCXT Server")
   - **Server URL**: URL сервера (по умолчанию: `http://localhost:3001`)
   - **Authentication Token**: Токен для аутентификации (по умолчанию: `your-secret-token`)
   - **Timeout**: Таймаут запросов в миллисекундах (по умолчанию: 30000)
4. Выберите биржи или используйте `*` для всех бирж
5. Нажмите **Create Provider**

### 2. Редактирование существующего провайдера

#### В Data Provider Setup Widget:
1. В списке **Existing Providers** найдите CCXT Server провайдер
2. Нажмите кнопку **Edit** (иконка карандаша)
3. Измените необходимые параметры:
   - Server URL
   - Authentication Token
   - Timeout
   - Список бирж
   - Приоритет
4. Нажмите **Save** для сохранения изменений

#### В Data Provider Debug Widget:
1. Найдите CCXT Server провайдер в списке провайдеров
2. Нажмите кнопку **Edit** (иконка карандаша)
3. Измените параметры в появившихся полях:
   - **Name**: Имя провайдера
   - **Priority**: Приоритет провайдера
   - **Server URL**: URL CCXT сервера
   - **Authentication Token**: Токен для аутентификации
   - **Timeout**: Таймаут запросов в миллисекундах
   - **Exchanges**: Список поддерживаемых бирж
4. Нажмите **Save** для сохранения изменений

## Конфигурация

### Параметры CCXT Server провайдера

```typescript
interface CCXTServerConfig {
  serverUrl: string;    // URL сервера (обязательно)
  token?: string;       // Токен аутентификации
  timeout?: number;     // Таймаут запросов (по умолчанию: 30000ms)
}
```

### Пример конфигурации

```typescript
{
  type: 'ccxt-server',
  name: 'Main CCXT Server',
  exchanges: ['*'], // Все биржи
  config: {
    serverUrl: 'http://localhost:3001',
    token: 'your-secret-token',
    timeout: 30000
  }
}
```

## Запуск CCXT Express сервера

Перед использованием CCXT Server провайдера убедитесь, что сервер запущен:

```bash
# Запуск сервера
npm run server

# Запуск с автоперезагрузкой при изменениях
npm run server:dev
```

Сервер будет доступен по адресу: `http://localhost:3001`

## Тестирование

### 1. Тестовые компоненты

Для тестирования функциональности созданы специальные компоненты:

#### TestCCXTServerProvider
- Доступен по адресу: `http://localhost:8080/test-ccxt-server`
- Позволяет тестировать подключение к серверу
- Создает тестовый провайдер
- Показывает список существующих CCXT Server провайдеров

#### TestDebugWidgetCCXTServer
- Доступен по адресу: `http://localhost:8080/test-debug-ccxt-server`
- Тестирует редактирование CCXT Server провайдеров в Debug виджете
- Создает тестовый CCXT Server провайдер
- Включает полный Data Provider Debug Widget для тестирования

### 2. Проверка подключения

1. Откройте тестовый компонент
2. Введите URL сервера и токен
3. Нажмите **Test Connection** для проверки доступности сервера
4. При успешном подключении создайте провайдер кнопкой **Create Provider**

### 3. Тестирование редактирования в Debug виджете

1. Перейдите на страницу `/test-debug-ccxt-server`
2. Нажмите **Create Test CCXT Server Provider** для создания тестового провайдера
3. В Data Provider Debug Widget найдите созданный провайдер
4. Нажмите кнопку **Edit** (иконка карандаша)
5. Убедитесь, что появились поля для CCXT Server:
   - Server URL
   - Authentication Token
   - Timeout
6. Измените значения и нажмите **Save**
7. Проверьте, что изменения сохранились

## Интеграция с существующей архитектурой

CCXT Server провайдер полностью интегрирован с существующей системой:

- **Совместимость**: Использует те же интерфейсы и типы
- **Приоритеты**: Поддерживает систему приоритетов провайдеров
- **Автовыбор**: Работает с автоматическим выбором провайдера
- **Уведомления**: Интегрирован с системой уведомлений

## Преимущества CCXT Server

1. **🎯 Обход CORS**: Основное преимущество - решение проблем CORS в браузере
2. **Производительность**: CCXT выполняется на сервере с Node.js
3. **Безопасность**: API ключи не передаются в браузер
4. **Масштабируемость**: Один сервер может обслуживать множество клиентов
5. **Кэширование**: Сервер кэширует CCXT инстансы и данные рынков
6. **Гибкость**: Сервер может быть запущен где угодно

## Устранение неполадок

### Проблемы с подключением

1. Убедитесь, что CCXT Express сервер запущен
2. Проверьте URL сервера (по умолчанию: `http://localhost:3001`)
3. Проверьте токен аутентификации
4. Проверьте настройки CORS

### Ошибки аутентификации

1. Убедитесь, что токен указан правильно
2. Проверьте переменную окружения `API_TOKEN` на сервере
3. По умолчанию токен: `your-secret-token`

### Таймауты

1. Увеличьте значение timeout в настройках провайдера
2. Проверьте стабильность сетевого соединения
3. Убедитесь, что сервер отвечает на health check: `http://localhost:3001/health`
